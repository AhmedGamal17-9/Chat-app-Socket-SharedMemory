Chat-app-Socket-SharedMemory

Professional Documentation for the high-performance Windows desktop chat application.

1. Quick Start

To get up and running immediately:

  1. CLONE: Retrieve the repository.

    git clone https://github.com/AhmedGamal17-9/Chat-app-Socket-SharedMemory.git
  2. PREPARE: Ensure MSYS2 UCRT64 toolchain is installed.

  3. BUILD: Run the build script in the project root. 

    build.bat

  4. RUN: Execute the binary.

    ChatSuite.exe

2. Prerequisites

Ensure your environment meets the following specifications before deployment or development.

  • Windows 10 or 11 (64-bit recommended)
  • MSYS2 (Win64) with UCRT64 environment

Required Packages

pacman -S mingw-w64-ucrt-x86_64-gcc mingw-w64-ucrt-x86_64-make

CRITICAL REQUIREMENT: You MUST add a "Folder Exclusion" in Windows Defender for the project directory. The application performs raw socket and memory data operations which may trigger false positives.

3. Development Setup

  1. Install MSYS2: Download from msys2.org.
  2. Initialize Environment: 
    pacman -Syu
  3. Install Compiler: 
    pacman -S mingw-w64-ucrt-x86_64-gcc
  4. Navigate to Source: 
    cd /c/Projects/Chat-app-Socket-SharedMemory

4. Build Instructions

Automated build using batch script: 

build.bat

Execute batch script from PowerShell: 

.\build.bat

Manual compilation via G++: 

g++ -std=c++17 -municode -mwindows src/*.cpp -o ChatSuite.exe -lws2_32 -lgdi32 -lcomctl32 -static-libgcc -static-libstdc++

Flags Explained

  • -mwindows: GUI application (no console)
  • -municode: Full Unicode support
  • -static-lib...: Portability (no DLL dependencies)

5. Runtime Usage

Mode 1: Socket Mode (LAN)

Server Setup (Host)

  1. Run ChatSuite.exe -> Select "Socket Mode"
  2. Role: "Server", Port: 8888
  3. Click "Start" (Verify log: Server listening...)

Client Setup (Remote)

  1. Run ChatSuite.exe -> Select "Socket Mode"
  2. Role: "Client", IP: Server IP, Port: 8888
  3. Click "Connect"

Mode 2: Shared Memory (Local IPC)

Ideal for secure, offline chat between two users on the SAME machine.

  1. User A (Person A): Select "Shared Memory Mode" -> Click Start.
  2. User B (Person B): Select "Shared Memory Mode" -> Click Start.
  3. Use the "Active Writer" toggle to switch input control.

11. Testing

Test A: Loopback Socket

Launch Server and Client both on 127.0.0.1 to test TCP stack.

Test B: Concurrency Stress

Open two shared memory instances. Spam send on one while typing on the other to verify mutex/semaphore locking.

12. Security & Antivirus

IMPORTANT:

False positives may occur with Heuristic engines (Generic.Malware). Add an exclusion for the build folder.

Network Encryption: Current version is PLAINTEXT. Do not send PII over public networks.

13. Release & Distribution

  1. Static Linking: Mandatory -static-libgcc -static-libstdc++.
  2. Compression: Zip the .exe (~2MB).
  3. Signing: Recommended for avoiding "Unknown Publisher".

16. FAQ

Is internet chat supported without VPN?

Generally no. Socket mode requires direct line of sight (LAN or Port Forwarding).

Why is the file size ~2MB?

Static linking of standard libraries causes this, but ensures portability without installing redistributables.
